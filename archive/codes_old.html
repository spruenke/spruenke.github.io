One Sample t-Test Simulation

This function uses resampling techniques to simulate the one sample t-test. Specifically, non-parametric, parametric and wild bootstrap are used. The code contains a function which takes 5 arguments:

  • x: The data vector to be tested (sample)
  • mu.t: The target mean (under null hypothesis) (by default 0)
  • nboot: The number of bootstrapping simulations (by default 4000)
  • alpha: The significance level (by default 0.05)
  • direction: A character string specifying the direction of the test. By default this is equal, testing the null that the sample mean and target mean are equal. Other possible arguments are "greater" ("lower") which tests the hypothesis that the sample mean is significantly above (below) the target mean.

Welch-Test Simulation

This function uses resampling techniques to simulate the two sample Welch-corrected t-test. Specifically, permutation, non-parametric, parametric and wild bootstrap are used. The null hypothesis is that the two sample means are equal. The code contains a function which takes 4 arguments:

  • x.1, x.2: The data vectors to be tested against each other (sample)
  • nboot: The number of bootstrapping simulations (by default 1000)
  • alpha: The significance level (by default 0.05)

MPT-Asset-Allocation

This function optimizes asset allocation in the sense of Modern Portfolio Theory. Namely, the constrained mean-variance portfolio (minimize variance with respect to target return), the global minimum variance portfolio and the maximization of the Sharpe Ratio. This function assumes a risk-free rate of zero. The code contains a function which takes 7 arguments:

  • mu: The expected returns of the assets (typically sample based)
  • sigma: The expected variance-covariance matrix of the assets (typically sample based)
  • gamma: The investor's risk aversion (default is set to 1)
  • method: Specifies the optimization method, i.e.: Constrained Minimum Variance ("cmv" - is default), Global Minimum Variance ("gmv") or Sharpe Ratio Maximization ("sr")
  • mu.p: The target return (only necessary for the Constrained Miniumum Variance - default is NULL)
  • wealth: Current portfolio value or wealth to invest into the stocks
  • prices: Current prices of the stocks used

Estimation of Asset Returns

This function estimates asset returns based on the sample using different methods. It takes 4 arguments:

  • x: The data matrix (sample). Needs dimension T x N, where T is the number of total Observations (dates) and N the number of assets
  • M: The window size on which the estimation takes place (by default 60)
  • lam: Mathematical Parameter for Bayes-Stein-Shrinkage (default is 1)
  • method: String specifying whether arithmetic mean ("ar"), geometric mean ("gr") or Bayes-Stein-Shrinkage ("bs") is used
